github+hexo构建博客

Posted on 2018-05-02

前一段时间在腾讯云上租了一台机器，就琢磨着搭建一个博客玩玩。找到了不少解决方案，一开始找了个开源的springboot博客系统，因为我是搞java的，熟门熟路，没没花多少工夫，就整了个springboot+nginx+mysql的博客，用了一段时间发现很不方便，更重要的是，自由度很低，想要加个功能得费不少功夫。后来又听说wordpress很高大上，由于个人对php只知皮毛，干脆就用docker构建镜像，一套操作下来，docker-wordpress+docker-mysql构建的博客出炉。开开心心玩了一段时间，但有个问题，因为穷，所以就只有一台最低配置的机器，加上我经常瞎整，时不时就把机器搞崩溃了，结果就是好不容易写了几篇心得，又得重来。痛定思痛，决心找一个能托管的资源的博客系统，顺着这个方向，终于找到了github+hexo这个神奇的东西。个人认为，这是我遇到的最好的解决方案。

搭建过程中遇到了不少问题，特此记录下来，以后方便查看

创建项目


注意选中部分必须是你的github账号，否则无法访问，不区分大小写

准备环境

需要使用到git node.js和hexo

1. 安装git
下载地址
git是再熟悉不过的工具了，这里就不再赘述安装方法了，贴个官网下载地址，各种系统的都支持，安装完成之后
>使用 git -version 看看是否安装成功

2. 安装node.js
下载地址
Hexo是基于nodeJS环境的静态博客,很多服务都需要使用npm,安装非常简单，同样安装完之后

使用 node -v 看看是否安装成功

3. 安装hexo
hexo及后面涉及到的相关服务，都使用npm工具来安装，首先需要将github上面创建的项目clone到本地仓库。

进入github，找到刚才创建的项目，复制项目地址

  • 创建空文件夹如：github-page ,cd命令到该目录
  • git init初始化仓库
  • git clone https://github.com/MfkOTT/mfkott.github.io.git 克隆项目到仓库
  • 再创建放博客相关的空文件夹如：blog cd命令到该目录
  • 安装hexo npm i -g hexo
    安装完成后

    使用 hexo -v 看看是否安装成功

  • 初始化hexo hexo init 初始化完成后，可以看见下面的目录结构

  • 目录介绍
    node_modules：是依赖包
    public：存放的是生成的页面
    scaffolds：命令生成文章等的模板
    source：用命令创建的各种文章
    themes：主题
    _config.yml：整个博客的配置
    db.json：source解析所得到的
    package.json：项目所需模块项目的配置信息

    配置项目

  • 用编辑器打开_config.yml，找到deploy，进行如下配置

    repo即是你在github上创建的项目地址
  • 回到blog目录中，执行以下命令
    hexo clean
    hexo generate
    hexo server
    注：hexo 3.0把服务器独立成个别模块，需要单独安装：npm i hexo-server 。
  • 打开浏览器输入：http://localhost:4000
  • 大功告成

    上传Github

  • 安装hexo-deployer-git服务，以便将写好的文章部署到github服务器上并让别人浏览
    npm install hexo-deployer-git --save
  • 执行命令(建议每次都按照如下步骤部署)
    hexo clean
    hexo generate
    hexo deploy
  • 在浏览器中输入http://mfkott.github.io.github.io就可以看到你的个人博客啦
